# Reporting a vulnerability

If you have a GitHub account, please use [GitHub's Security Advisories page](https://github.com/samuel-lucas6/Geralt/security/advisories). Otherwise, please email `security at samuellucas (dot) com` using an informative subject line (e.g., `VULN: <problem> in Geralt`).

GitHub PRs, issues, and discussions that do not follow the security policy will be redacted, closed, and locked in favour of the above. See below for this rationale.

I will aim to respond as soon as possible. However, I am often busy doing other things and do not check GitHub/my emails every day. If you do not receive a reply within a week, assume that I am on holiday or that your email has been filtered out.

Please respect the [standard](https://projectzero.google/2020/01/policy-and-disclosure-2020-edition.html) 90 days or when a patch is released disclosure timeline. I have a life outside of open source projects and cannot necessarily dump everything to fix a problem.

Confirmed vulnerabilities will be disclosed by me via GitHub Security Advisories and in the release notes of the patched version (the `Security` section from [Keep a Changelog](https://keepachangelog.com/)).

I do not offer bug bounties at this time. However, I will add your name or username (with a link to a profile/website) to the [Acknowledgements](https://www.geralt.xyz/#acknowledgements) section of the website and in the release notes of the patched version as a thank you.
